# EchoCare

EchoCare 是一个课程作业用的移动端优先 PWA 原型，主题是“基于语音输入的情绪记录与压力感知助手”。它支持情绪日记、压力指数、关键词、情绪歌单、每日书籍/安慰语、一周趋势和用户体验测量。

## 运行方式

在项目目录运行：

```bash
npm test
npm run serve
```

然后在浏览器打开：

```text
http://localhost:4173
```

也可以直接用浏览器打开 `index.html` 查看界面，但语音识别、PWA 安装和 Service Worker 更适合通过本地服务器访问。

## 已实现功能

- 移动端 App 风格界面，桌面端自动显示为演示设备框。
- 语音输入入口：浏览器支持 `SpeechRecognition` 时可尝试识别，不支持时自动回退到文字输入。
- 本地情绪分析：识别焦虑、低落、疲惫、开心、平静。
- 压力指数：输出 0-100 分压力值。
- 关键词识别：学习任务、睡眠、人际关系、未来规划、身体状态、自我要求。
- 情绪陪伴推荐：根据情绪生成 4 首推荐歌曲和推荐理由。
- 每日书籍/安慰语：根据情绪推荐书籍，并生成原创安慰文字。
- 一周趋势：本地保存记录，生成压力柱状图、高频关键词和概览文字。
- 体验测量：生成可用性、语音满意度、隐私信任、情绪舒适度和综合评分。

## DeepSeek 接入建议

当前原型默认使用本地规则模拟 AI 分析，保证课堂展示时无需网络和密钥也能稳定运行。

如需接入 DeepSeek，不要把 API Key 写进前端。推荐结构：

```text
EchoCare PWA
  -> /api/analyze 本地后端或 Serverless 函数
      -> DeepSeek Chat Completions API
          -> 返回 JSON 格式分析结果
```

建议让后端返回如下结构：

```json
{
  "emotion": { "id": "anxious", "label": "焦虑", "tone": "需要放慢", "color": "#7c6be8" },
  "stress": 78,
  "keywords": ["学习任务", "睡眠"],
  "feedback": "你提到的学习任务正在占用较多心理能量。",
  "suggestion": "做 3 轮 4-6 呼吸，然后只写下明天最重要的一件小事。",
  "playlist": [
    { "title": "Weightless", "artist": "Marconi Union", "reason": "适合把注意力从焦虑拉回呼吸。", "mood": "舒缓" }
  ],
  "book": { "title": "被讨厌的勇气", "author": "岸见一郎、古贺史健", "reason": "适合在担心评价时阅读。" },
  "comfortText": "先把自己放回当下。今天不需要一次解决全部问题。"
}
```

## 作业汇报角度

这个原型可以被描述为一个独立运行的情绪陪伴模块，也可以嵌入校园心理健康平台或个人健康管理 App。课程汇报可重点展示：用户画像、语音记录任务流程、情绪反馈设计、陪伴推荐设计、SUS/满意度/信任感测量和一周趋势反馈。
